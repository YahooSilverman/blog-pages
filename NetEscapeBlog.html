It Begins

Initially, I had programmed a very basic text based game in C# that ran in console. However, while creating the enemy class I felt like I could make something a bit more unique. This is where the 90’s desktop/virus theme came into play. To start, I create the image using quick Google searches of Windows 98 search bars and icons as well as application shots of AOL Instant Messenger and Windows Media Player, I’m not sure if the timeline lines up completely, so for lore purposes let’s just pretend. To add some character, I grabbed a sweet Sailer moon picture for the wallpaper, a Bubsy 3D icon for the AIM user and a photo of that computer virus from the first season of Buffy the Vampire Slayer.

After building my canvas in unity, I added separate images for the Media Player and AIM application. I created three text features, one is named enemyText and simply prints the enemy’s output within the AIM console, with a second to display the users input which comes from the third text option which was created using an input field. Finally, I created four buttons (the send button for AIM, the play button for the Media Player, and a desktop icon for each). I simply copied the button images in Paint 3D and saved them as independent images to use as decals for each button for “seamless” blending. Now for the fun part.

The first script I wrote, titled OpenPrograms, is a public class using MonoBehavior. This class simply runs a method titled OpenPanel and checks if Panel != null, in which case we use the SetActive(true) feature. I then of course attach this script to my AIM desktop icon and use the On Click feature to open the panel. I also use this script for the media player. Next up, I built an audioController that is responsible for playing audio upon request and will also shoot back a LogWarning if the name of the file is not found. I built another class called textTransfer that runs a Coroutine. In the Coroutine we run a WaitForSeconds followed by printing the first message from the character named simply “Virus”. Open showing the message we play an audio clip that sounds as close to an AIM sound as I could find on Itch, and let’s be honest, anyone with that sick wallpaper would almost certainly have a bespoke message alert tone anyway. The other two buttons (send message and play music) are built similarly to the previous two, but rather than opening a window they complete an action. The Media Player button starts playing a weird funk song I found on Itch, and the send message button takes the text entered in the input field and prints it in the AIM display under userText.

Next weekend I plan on rewriting that text-based game to play in the AIM console and add features such as screen shake (can be done by building a script attached to the camera) and adding some cool sound effects.
